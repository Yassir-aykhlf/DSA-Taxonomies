# Greedy Algorithms

> Build a solution by making the locally optimal choice at each step, trusting that these choices compose into a global optimum.

> **Use when**
> - Sorting by a single key (end time, ratio, value) and sweeping left to right yields the answer.
> - You schedule or merge intervals: pick the earliest-finishing, count overlaps, cover a range.
> - An exchange argument shows any optimal solution can be reshaped toward your greedy pick without loss.
> - You construct an answer incrementally — digit by digit, character by character — discarding suboptimal prefixes.
> - You always service the highest-priority item next (often via a heap) and never reconsider.

> **Complexity**
> - Sort-then-sweep: **O(n log n)** time (the sort dominates), **O(1)–O(n)** space.
> - Single linear scan with a running min/max or counter: **O(n)** time, **O(1)** space.
> - Heap-backed greedy (priority processing): **O(n log n)** time, **O(n)** space.

## Related patterns

- **Dynamic Programming** — when greedy choices fail, DP explores all options; greedy is the faster path when it holds.
- **Heap & Priority Queue** — supplies the "best available next" element in priority-driven greedy.
- **Sorting** — most greedy proofs start from a sorted order on some key.
- **Interval problems** — sweep-line and scheduling greedies operate directly on intervals.
