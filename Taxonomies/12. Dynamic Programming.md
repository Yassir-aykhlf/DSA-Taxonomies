# Dynamic Programming

> Solve a problem by combining answers to overlapping subproblems, storing each result once so it is never recomputed.

> **Use when**
> - Subproblems overlap and the optimal answer is built from optimal sub-answers (optimal substructure).
> - You count ways, or optimize a max/min, over sequences, grids, or intervals (LIS, LCS, edit distance, path sums).
> - Each step is a take-or-leave / choose-an-option decision (knapsack, stock trading, coin change).
> - A brute-force recursion repeats the same calls — add memoization or fill a table bottom-up.
> - State fits in a few dimensions or a bitmask (subset-of-visited, assignment problems, TSP).

> **Complexity**
> - Time is (number of states) × (transition cost): 1-D linear **O(n)**, 2-D grid/string **O(n·m)**, interval **O(n^3)**.
> - Space equals the table size, often reducible to **O(n)** or **O(1)** with rolling arrays.
> - Bitmask DP over n items: **O(2^n · n)** states-and-transitions.

## Related patterns

- **Recursion** — DP is recursion plus a cache; most tables start as a recursive relation.
- **Greedy Algorithms** — when a local choice is provably optimal, greedy replaces a full DP.
- **Backtracking** — exhaustive search that becomes DP once overlapping subproblems are memoized.
- **Divide & Conquer** — both split into subproblems, but DP reuses overlapping ones.
