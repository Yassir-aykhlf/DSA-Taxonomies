# Prefix Sum

> Precompute cumulative values so any range query collapses to a constant-time difference of two prefixes.

> **Use when**
> - You answer many range-sum (or range-product/XOR) queries over a static array or matrix.
> - You count or find subarrays whose sum equals, or is divisible by, a target (pair prefixes via a hashmap).
> - You need a pivot/equilibrium index where the left sum equals the right sum.
> - You apply many range updates then read once (difference array, the inverse of prefix sum).
> - You track a running min/max/GCD or prefix XOR alongside the scan.

> **Complexity**
> - Build the prefix array: **O(n)** time and space; each range query is then **O(1)**.
> - Subarray-sum-equals-k via hashmap of prefix counts: **O(n)** time, **O(n)** space.
> - 2D prefix sum: **O(m · n)** build, **O(1)** per rectangle query.

## Related patterns

- **Sliding Window** — an alternative for some contiguous range-sum problems.
- **Hash Table** — paired with prefix sums to count subarrays in one pass.
- **Dynamic Programming** — prefix sums can optimize certain DP transitions.
- **Segment Tree** — an alternative for mutable range queries.
