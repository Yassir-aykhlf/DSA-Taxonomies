# Trie

> A prefix tree that stores strings character-by-character along shared paths, so prefix and word lookups run in O(length) regardless of how many keys are stored.

> **Use when**
> - You query many strings against a shared set of prefixes: autocomplete, search suggestions, dictionary word lookup.
> - You need wildcard or partial matching (`.` placeholders, prefix-and-suffix, camelcase patterns).
> - You walk a grid or stream and want to prune dead branches as soon as a prefix can't continue.
> - You maximize XOR over numbers by treating their bits as a path in a binary trie.
> - You count or rank words by a shared prefix, or detect when one word contains another as a prefix.

> **Complexity**
> - Insert / search / startsWith a key of length L: **O(L)** time.
> - Space: **O(total characters inserted · alphabet)** in the worst case (array-backed children).

## Related patterns

- **String Manipulation** — tries excel at prefix-heavy string problems.
- **Tree Traversal** — DFS/BFS techniques apply directly to walking a trie.
- **Backtracking** — pairs with a trie to prune grid/word searches (Word Search II).
- **Hash Table** — an alternative to tries for some string-lookup problems.
- **Bit Manipulation** — binary tries solve maximum-XOR problems bit by bit.
