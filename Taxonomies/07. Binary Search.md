# Binary Search

> Repeatedly halve a sorted or monotonic search space to locate a value — or the boundary between "no" and "yes" — in O(log n).

> **Use when**
> - The array is sorted, or there is a monotonic predicate (once true, stays true) you can test at any point.
> - You want a boundary: first/last index satisfying a condition, an insertion point, a lower/upper bound.
> - "Search on the answer": the answer lies in a numeric range and feasibility is monotonic — minimize a maximum, or maximize a minimum.
> - A rotated or mountain array, where a local comparison reveals which half to discard.

> **Complexity**
> - Search over n elements: **O(log n)** time, **O(1)** space.
> - Search on the answer over a value range R with an O(n) feasibility check: **O(n · log R)**.
