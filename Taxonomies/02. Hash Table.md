# Hash Table

> Trade space for time: store seen keys, counts, or groups so lookup, insert, and delete run in O(1) average.

> **Use when**
> - You need to test membership or spot duplicates in one pass (have I seen this value before?).
> - You count occurrences and act on frequency: anagrams, top-k, majority, character signatures.
> - You match a complement or a running prefix sum on the fly (two-sum, subarray-sum-equals-k).
> - You group items by a derived key (sorted string, shifted pattern, hashed signature).
> - You map one domain to another and must keep the mapping consistent (isomorphic strings, word pattern).
> - You need an O(1) keyed structure to back a cache or custom design (LRU/LFU, HashMap).

> **Complexity**
> - Lookup / insert / delete: **O(1)** average, **O(n)** worst case (adversarial collisions).
> - Single-pass scan with a map: **O(n)** time, **O(n)** space for the keys stored.
> - Sorting-key grouping (e.g. Group Anagrams): **O(n · k log k)** time for strings of length k.

- **Two Pointers** — offers space-time trade-offs for duplicate and pair checks.
- **Sliding Window** — pairs naturally with hash maps for frequency tracking.
- **Trie** — an alternative to hashed string signatures.
- **Arrays** — in-place index hashing avoids extra structures.
