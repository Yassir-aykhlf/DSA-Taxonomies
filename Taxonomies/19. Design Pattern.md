# Design

> Build custom data structures and systems that expose a clean API while meeting strict per-operation performance constraints.

> **Use when**
> - The problem says "Design …" / "Implement …" and you must support a set of methods with target time bounds.
> - You need O(1) or O(log n) get/put under an eviction policy (LRU, LFU, time-based caching).
> - You implement a container from scratch — hash map/set, stack via queues, circular queue, iterator.
> - You model a stateful system: calendar booking, rate limiter, parking lot, browser history, a game.
> - You combine structures (hash map + linked list, hash map + heap) to hit two constraints at once.

> **Complexity**
> - LRU/LFU cache get and put: **O(1)** with a hash map plus a doubly linked list.
> - Trie insert/search: **O(L)** in key length; calendar/booking with a balanced structure: **O(log n)** per op.
