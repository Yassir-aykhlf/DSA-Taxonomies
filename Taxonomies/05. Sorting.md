# Sorting

> Arrange data into order so that searching, grouping, and greedy choices become easy — as an end in itself or a setup step.

> **Use when**
> - Order unlocks the rest: a sorted array enables binary search, two-pointer sweeps, or dedup.
> - You need the k-th element or top-k without fully sorting (quickselect, heap, bucket).
> - Keys have a small bounded range, so counting / radix / bucket beats comparison sorts.
> - Ordering needs a custom rule (largest-number concatenation, height reconstruction, log files).
> - You process intervals or run a greedy strategy that only works once items are sorted.

> **Complexity**
> - Comparison sort (merge / quick / heap): **O(n log n)** time; merge needs **O(n)** space, heap **O(1)**.
> - Counting / radix sort over range k or d digits: **O(n + k)** / **O(d · (n + b))** time, **O(n + k)** space.
> - Quickselect (k-th element): **O(n)** average, **O(n²)** worst; heap top-k: **O(n log k)**.

## Related patterns

- **Divide & Conquer** — recursive partitioning and merging.
- **Heap** — priority-ordered extraction for selection tasks.
- **Two Pointers** — in-place partitioning during quicksort and the merge sweep.
- **Greedy** — sorted order as a prerequisite for choices.
