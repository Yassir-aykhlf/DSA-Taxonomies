# Union Find

> A disjoint-set structure that tracks which elements share a group, answering "are these connected?" and merging groups in near-constant amortized time.

> **Use when**
> - You repeatedly merge groups and ask whether two elements are now connected (dynamic connectivity).
> - You count connected components, or detect a cycle by finding an edge whose endpoints already share a root.
> - You build a minimum spanning tree with Kruskal: sort edges, union endpoints that aren't yet joined.
> - You group elements by an equivalence relation: merged accounts, equal-variable equations, similar strings.
> - Edges or unions arrive incrementally (islands appearing, edges added over time) and a static graph scan won't do.

> **Complexity**
> - With path compression + union by rank: **O(α(n))** amortized per find/union (α = inverse Ackermann, effectively constant).
> - Space: **O(n)** for the parent and rank/size arrays.

## Related patterns

- **Graph Algorithms** — union-find excels at connectivity and component questions.
- **Minimum Spanning Tree** — Kruskal's algorithm is built on union-find.
- **DFS/BFS** — an alternative way to find components on a static graph.
- **Sorting** — Kruskal sorts edges before unioning; threshold problems sort queries.
