# Segment Tree & Fenwick Tree

> Tree-based structures that answer range queries and apply updates on an array in O(log n) per operation.

> **Use when**
> - You need repeated range aggregates (sum, min, max, GCD) on an array whose values keep changing.
> - Point updates interleave with range queries, so a static prefix-sum array is not enough.
> - You must apply updates to a whole range at once — lazy propagation defers the work.
> - You count inversions, smaller-after-self, or reverse pairs by querying ranks as you scan.
> - You process intervals/coordinates (skyline, calendars, rectangle area) with coordinate compression.

> **Complexity**
> - Build: **O(n)**; point update and range query: **O(log n)** each.
> - Range update with lazy propagation: **O(log n)**; space **O(n)** (segment tree ~2n-4n, Fenwick exactly n+1).
